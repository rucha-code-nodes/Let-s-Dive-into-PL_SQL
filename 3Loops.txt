# ✅ 1) SIMPLE LOOP (Basic LOOP)

### 🔥 Meaning:

It runs **again and again** until you explicitly stop it using:

✅ `EXIT;`
or
✅ `EXIT WHEN condition;`

### Syntax

```sql
LOOP
   statements;
   EXIT WHEN condition;
END LOOP;
```

---

## Example (Yardi style): Print rent receipt for 12 months

```sql
DECLARE
   v_month NUMBER := 1;
BEGIN
   LOOP
      DBMS_OUTPUT.PUT_LINE('Generating rent receipt for Month = ' || v_month);

      v_month := v_month + 1;

      EXIT WHEN v_month > 12;
   END LOOP;
END;
/
```

### Output:

Month 1 receipt
Month 2 receipt
...
Month 12 receipt

### ⭐ Interview Point:

* Used when you **don’t know exact iterations** OR want manual control.
* Must have EXIT otherwise infinite loop.

# ✅ 2) WHILE LOOP

### 🔥 Meaning:

It checks condition **first**, then runs loop.
If condition is false initially → loop will not run even once.

### Syntax

```sql
WHILE condition LOOP
   statements;
END LOOP;
```

---

## Example (Yardi style): Apply late fee until pending balance becomes 0

```sql
DECLARE
   v_balance NUMBER := 5000;
BEGIN
   WHILE v_balance > 0 LOOP
      DBMS_OUTPUT.PUT_LINE('Pending Balance = ' || v_balance);

      v_balance := v_balance - 1000;
   END LOOP;

   DBMS_OUTPUT.PUT_LINE('Balance cleared!');
END;
/
```

### Output:

Pending Balance = 5000
Pending Balance = 4000
...
Balance cleared!

### ⭐ Interview Point:

* Used when repetition depends on a **condition**.
* Condition is checked at the start.

---

# ✅ 3) FOR LOOP (Most common in Interviews)

### 🔥 Meaning:

Used when you know **exact number of iterations**.
PL/SQL automatically increments the counter.

### Syntax

```sql
FOR i IN 1..10 LOOP
   statements;
END LOOP;
```

---

## Example (Yardi style): Calculate rent for 12 months

```sql
DECLARE
   v_rent NUMBER := 15000;
BEGIN
   FOR m IN 1..12 LOOP
      DBMS_OUTPUT.PUT_LINE('Month ' || m || ' Rent = ' || v_rent);
   END LOOP;
END;
/
```

### ⭐ Interview Point:

* Best for fixed cycles like **12 months rent, yearly lease, etc.**
* Counter variable (m) is auto-managed (no need to increment).

---

# ✅ 4) EXIT WHEN (Very Important)

### 🔥 Meaning:

Used to stop loop when a condition becomes true.

---

## Example (Yardi style): Stop when rent exceeds limit

```sql
DECLARE
   v_rent NUMBER := 10000;
BEGIN
   LOOP
      DBMS_OUTPUT.PUT_LINE('Rent = ' || v_rent);

      v_rent := v_rent + 2000;

      EXIT WHEN v_rent > 20000;
   END LOOP;

   DBMS_OUTPUT.PUT_LINE('Rent limit exceeded, stopped!');
END;
/
```

### ⭐ Interview Point:

* Used mostly with **simple LOOP**
* Prevents infinite loop

---

# ✅ BEST INTERVIEW EXAMPLE (Real Yardi Type)

### Scenario:

We have tenant rent. Increase rent 5% every year for 5 years.

```sql
DECLARE
   v_rent NUMBER := 20000;
BEGIN
   FOR yr IN 1..5 LOOP
      v_rent := v_rent + (v_rent * 0.05);

      DBMS_OUTPUT.PUT_LINE('Year ' || yr || ' Updated Rent = ' || v_rent);
   END LOOP;
END;
/
```

### ⭐ This is perfect for Yardi because:

* Rent escalation is real estate common
* Shows calculation + loop use

# 🔥 Quick Difference Table (Must Remember)

| Loop Type  | Condition Check        | Best Use                                |
| ---------- | ---------------------- | --------------------------------------- |
| LOOP       | Inside loop using EXIT | Unknown iterations                      |
| WHILE LOOP | At beginning           | Run until condition becomes false       |
| FOR LOOP   | Fixed range            | Known iterations (months/years/records) |
